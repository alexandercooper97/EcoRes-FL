04 Algoritmos
FedAvg
Baseline clásico. Selección aleatoria de clientes, agregación ponderada por datos locales. Gradientes completos.
McMahan et al., 2017
FedProx
Añade penalización proximal μ‖w − w_global‖² para controlar la divergencia en entornos non-IID.
Li et al., 2020
CarbonAware
Selección por score multiobjetivo que minimiza carbono instantáneo. Prefiere clientes con energía renovable.
Carbon-aware scheduling
EcoRes-FL ★
ILP binario + proximal + top-k sparsification. Reduce comunicación ≥70% con pérdida marginal de accuracy.
Propuesto · 2025
Flujo del simulador
# Por cada ronda r = 1..R:
select_clients(method, profiles, r)     # selección según algoritmo
  └─ [FedAvg/FedProx]  → aleatorio uniforme
  └─ [CarbonAware]     → rank por score ponderado
  └─ [EcoRes-FL]       → ILP (scipy.milp) o greedy fallback

for each client_k in selected:
    W_k, b_k = local_train_softmax(X_k, W, b, μ)   # SGD local
    dW, db   = sparsify_update(W_k-W, b_k-b, k%)   # top-k (EcoRes)
    estimate_cost(profile_k, density, ci_k)          # energía + carbono

W += Σ_k (n_k/N) · dW_k                             # FedAvg aggregation
evaluate(X_test, W)                                  # accuracy + F1
05 Cómo desplegar en GitHub Pages

Esta app es un archivo HTML único. No necesita servidor, base de datos ni build step. Se puede alojar gratis en GitHub Pages y cualquier usuario puede usarla desde su navegador.

1
Crear el repositorio
Crea un repo público en GitHub con el nombre ecores-fl y sube el archivo index.html a la rama principal.
2
Activar GitHub Pages
Settings → Pages → Source: main branch, carpeta / (root) → Save.
3
Acceder al enlace
En ~60 segundos tu app estará en https://tu-usuario.github.io/ecores-fl y compartible con cualquiera.
4
(Opcional) Dominio propio
En Settings → Pages → Custom domain: añade tu dominio. Activa "Enforce HTTPS".
Comandos Git
# Clonar / crear repo local
git init ecores-fl
cd ecores-fl
cp /ruta/a/index.html .

# Subir a GitHub
git add index.html
git commit -m "feat: add EcoRes-FL web app"
git remote add origin https://github.com/user/ecores-fl.git
git push -u origin main

# La URL de tu app será:
https://user.github.io/ecores-fl
¿Por qué HTML puro y no Streamlit?
Streamlit necesita un servidor Python activo (RAM ~500MB), se duerme tras inactividad, y no escala a múltiples usuarios simultáneos. Un HTML estático se sirve desde un CDN global, carga en <1s y soporta tráfico ilimitado sin costo.
